# Architecture Decision Records (ADR)

## Purpose

Track significant architecture and design decisions in a durable, chronological format.

Use ADRs when a decision:

- affects multiple components
- changes operational risk
- introduces long-term constraints

---

## Naming

Use numeric prefix with short title:

- `0001-auth-provider-choice.md`
- `0002-memory-type-normalization.md`

---

## Required Sections

- Status (`proposed`, `accepted`, `deprecated`, `superseded`)
- Context
- Decision
- Consequences
- Alternatives considered

---

## Relationship to Other Docs

- `docs/decisions.md` can summarize decisions
- ADR files store full rationale and trade-offs
