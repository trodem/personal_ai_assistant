# Security Policy

## Supported Versions

This project is currently in active MVP development.

Security fixes are applied to the default branch only.

| Version | Supported |
| --- | --- |
| `main` | :white_check_mark: |
| older branches | :x: |

## Reporting a Vulnerability

Please report security issues privately. Do not open public issues for vulnerabilities.

Preferred process:

1. Send a report to: `security@personal-ai-assistant.local` (replace with your real security email).
2. Include:
- affected component/file
- reproduction steps
- impact assessment
- suggested fix (if available)
3. You will receive an acknowledgement within 72 hours.
4. We will provide status updates until remediation.

## Disclosure Policy

- We follow responsible disclosure.
- We aim to patch critical issues before public disclosure.
- After a fix is available, we may publish a brief advisory/changelog note.

## Scope

In-scope:

- authentication bypass
- cross-user data access
- secret/token leakage
- file upload/storage security issues
- injection vulnerabilities

Out-of-scope:

- theoretical issues without a reproducible path
- low-impact informational findings without exploitation path

## Safe Harbor

If you act in good faith and follow this policy, we will not pursue legal action for your research.
